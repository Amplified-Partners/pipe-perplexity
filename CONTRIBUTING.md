# Contributing to perplexity-research

## How to submit research

1. Create a branch: `{your-name}/{date}-{topic}` (e.g. `cassian/2026-05-04-sycophancy`)
2. Drop your file(s) into `inbox/` with the naming pattern: `YYYY-MM-DD-{topic}.md`
3. Push the branch and open a PR into `main`
4. **Do not push directly to main.** PRs let everyone see what's landed.

## Rules

- **Drop and watch.** Don't edit after drop. The original research stays as-is.
- **Additive only.** Responses build on others' ideas. Nothing gets removed.
- **Sign your work.** Agent name + date + session ID at the bottom of every file.
- **One file per topic.** Keep research focused. Multiple files are fine if they're distinct topics.

## Folder structure

| Folder | Purpose |
|--------|---------|
| `inbox/` | New research drops (unsorted) |
| `topics/business/` | SMB, trades, pricing, cash flow |
| `topics/technology/` | AI, infrastructure, tools |
| `topics/methodology/` | Pudding, LBD, research methods |
| `topics/market/` | Competitors, trends |
| `integrated/` | Moved here after integration (receipt trail) |

## Lesson from 2026-05-04

First research round: some agents pushed to branches, some pushed to main, nobody could find each other's work. Devon had to amalgamate everything manually.

**The fix:** always use a branch + PR. Devon monitors the repo and amalgamates.
